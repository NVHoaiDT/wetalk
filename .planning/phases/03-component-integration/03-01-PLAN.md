Edit src/testing/setup-tests.ts and add the following mock block (after existing setup):

```typescript
// Mock i18next for component tests
import { vi } from 'vitest';

vi.mock('react-i18next', () => ({
  useTranslation: () => ({
    t: (key: string, defaultValue?: string) => defaultValue || key,
    i18n: {
      language: 'en',
      changeLanguage: vi.fn().mockResolvedValue(undefined),
      exists: vi.fn().mockReturnValue(true),
    },
  }),
  initReactI18next: {
    type: '3rdParty',
    init: vi.fn(),
  },
  I18nextProvider: ({ children }: { children: React.ReactNode }) => children,
}));
```

This mock:

- Returns key as fallback (e.g., t('post.action.create') returns 'post.action.create')
- Enables language switching in tests (changeLanguage is mockable)
- Prevents file I/O (no actual JSON loading during tests)
- Isolates component logic from i18n system

All vitest component tests will use this mock unless explicitly over-ridden (not expected for Phase 3).
