Create `src/i18n/config.ts` with the following configuration:

```typescript
import i18next from 'i18next';
import LanguageDetector from 'i18next-browser-languagedetector';
import { initReactI18next } from 'react-i18next';

// English namespace (will be extended with VI translations in Phase 5)
const enCommon = {
  // Namespace structure placeholder for now (will be populated in Phase 2)
};

const resources = {
  en: {
    common: enCommon,
  },
  vi: {
    // Vietnamese namespace (will be added in Phase 5)
    common: {},
  },
};

i18next
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources,
    fallbackLng: 'en',
    defaultNS: 'common',
    interpolation: {
      escapeValue: false, // React already escapes XSS
    },
    detection: {
      order: ['localStorage', 'navigator'],
      caches: ['localStorage'],
    },
  });

export default i18next;
```

**Explanation:**
- **LanguageDetector**: Detects browser language from `navigator.language`, falls back to localStorage if user previously set preference
- **fallbackLng: 'en'**: Default to English if detection fails
- **defaultNS: 'common'**: All keys use `common` namespace by default (Phase 3 will add other namespaces like `posts`, `messages`)
- **detection.order**: Check localStorage first (user preference), then navigator (browser locale)
- **interpolation.escapeValue: false**: React handles XSS protection; i18next shouldn't double-escape

This structure supports feature-based namespaces (posts.json, messages.json, etc.) which will be added in Phase 2.

Create `src/lib/i18n-store.ts` with Zustand store for language state management:

```typescript
import { create } from 'zustand';
import { persist } from 'zustand/middleware';
import i18next from '@/i18n/config';

type LanguageStoreState = {
  language: 'en' | 'vi';
  setLanguage: (lang: 'en' | 'vi') => void;
};

export const useLanguageStore = create<LanguageStoreState>()(
  persist(
    (set) => ({
      language: 'en',
      setLanguage: (lang: 'en' | 'vi') => {
        i18next.changeLanguage(lang);
        set({ language: lang });
      },
    }),
    {
      name: 'language-store', // localStorage key
      version: 1,
    }
  )
);
```

**Explanation:**
- **Zustand create()**: Creates a store hook following WeTalk's existing pattern (see src/lib/auth.tsx)
- **persist middleware**: Automatically saves `language` state to localStorage under key `language-store`
- **language: 'en' | 'vi'**: Type-safe language options (only English and Vietnamese allowed)
- **setLanguage()**: Updates store AND calls `i18next.changeLanguage()` to trigger component re-renders with new language
- **version: 1**: For future migration if store schema changes

This store will be initialized on app startup and merged with i18next's browser language detector:
1. App loads → LanguageDetector checks localStorage for `language-store`
2. If found, i18next loads that language
3. Zustand store hydrates with same language from localStorage
4. User changes language → setLanguage() updates both Zustand and i18next

**Pattern matches existing WeTalk auth store** in src/lib/auth.tsx for consistency.

```typescript
// At the end of src/i18n/config.ts, add:

declare module 'react-i18next' {
  interface CustomTypeOptions {
    defaultNS: 'common';
    resources: {
      common: typeof resources['en']['common'];
    };
  }
}

declare global {
  interface Window {
    i18next: typeof i18next;
  }
}
```

**Explanation:**
- **CustomTypeOptions interface**: Tells TypeScript about your translation structure
- **defaultNS: 'common'**: All t() calls use common namespace by default
- **resources structure**: TypeScript infers available keys from the actual resource definition
- When developers use `t('unknown.key')`, TypeScript will report an error at compile-time if the key doesn't exist in resources

This prevents untranslated string bugs (flagged in PITFALLS research as HIGH severity).

Note: Phase 2 string extraction will regenerate this interface with actual keys from the codebase.
