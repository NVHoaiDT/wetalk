Create `.i18nextrc.json` file to configure i18next-scanner. This configuration tells the scanner what files to scan and where to output results.

**Configuration:**

```json
{
  "input": ["src/**/*.${ext}", "src/**/*.ts", "src/**/*.tsx"],
  "output": "./",
  "options": {
    "debug": true,
    "func": {
      "list": ["t"]
    },
    "lngs": ["en"],
    "ns": [
      "common",
      "post",
      "message",
      "community",
      "auth",
      "dashboard",
      "notifications",
      "search",
      "profiles",
      "users",
      "chatbot",
      "settings",
      "postComments"
    ],
    "defaultLng": "en",
    "defaultNs": "common",
    "resource": {
      "loadPath": "public/locales/{{lng}}/{{ns}}.json"
    }
  }
}
```

**Key settings:**

- **input:** Scan all `.ts` and `.tsx` files in `src/`
- **output:** Write results to current directory
- **func.list:** [`"t"`] — scanner looks for `t('key')` calls (react-i18next pattern)
- **lngs:** [`"en"`] — we're validating English namespace only
- **ns:** List all 13 namespace files (exact names: common, post, message, community, auth, dashboard, notifications, search, profiles, users, chatbot, settings, postComments)
- **resource.loadPath:** Points to our JSON file location (`public/locales/{{lng}}/{{ns}}.json`)

**Rules:**

- Save file as `.i18nextrc.json` in workspace root
- Use exact namespace names matching JSON filenames
- Ensure valid JSON (no trailing commas)

## Terminology & Tone Guidance

### Platform-specific terms:
- **Post** → Bài đăng (not "tin nhắn" which means message)
- **Community** → Cộng đồng
- **Message** → Tin nhắn (one-on-one private message, not public post)
- **Share** → Chia sẻ (not "phân phối")
- **Like/Reaction** → Thích
- **Comment** → Bình luận
- **Follow** → Theo dõi
- **Notification** → Thông báo
- **Profile** → Hồ sơ

### Tone examples:
- "Loading..." → "Đang tải..." (friendly, not formal)
- "This action cannot be undone." → "Hành động này không thể hoàn tác." (clear, simple)
- "No results found" → "Không tìm thấy kết quả" (direct, no apology needed)

### Technical considerations:
- Avoid abbreviations unless they're platform-standard
- Keep field labels parallel structure (adjective + noun pattern consistent)
- For validation, be specific: "Email is invalid" vs vague "Invalid input"

## Text Length Considerations

**Vietnamese is typically 20-30% longer than English.**

- Short English strings (5-10 chars): May expand to 8-15 chars in Vietnamese
- Medium strings (10-30 chars): Expect 30-40% expansion
- Long strings (30+ chars): Usually 20-25% expansion

### Examples:
| English | Vietnamese | Expansion |
|---------|---|---|
| "Delete Post?" (12 chars) | "Xóa bài đăng? " (14 chars) | ~16% |
| "Confirm" (7 chars) | "Xác nhận" (8 chars) | ~14% |
| "Failed to create post. Please try again." (40 chars) | "Không thể tạo bài đăng. Vui lòng thử lại." (44 chars) | ~10% |

**UI consideration:** The app uses flexible Tailwind widths, so text length expansion is generally not a problem. However, be mindful of very long Vietnamese strings in buttons or modals (truncation may occur in tight layouts).
